; ln_FNVLinkEvtAid.txt
; YesMan AI Live Link — aid-item-used event (JIP). ISOLATED in its own ln_ script ON PURPOSE so a
; compile/registration hiccup here can't take down the confident base events in ln_FNVLinkEvt.txt.
;
; COVERAGE: SetOnUseAidItemEventHandler REQUIRES an item/FormList filter — there is NO "any
; ingestible" form, and OMITTING the filter is a compile error ("Missing parameter", verified
; in jip_ln_nvse.log). So we register ONE shared handler (a stored lambda) across EVERY ALCH
; (ingestible) defined by FalloutNV.esm — 189 items: meds, chems, food, drink, alcohol, poisons,
; plus ALCH-typed specials (Stealth Boy, Weapon Repair Kit, NCR Emergency Radio, skill
; MAGAZINES). Items are referenced by EditorID (load-order-independent; FalloutNV.esm is always
; loaded). DLC / mod-added aid can be appended later the same way. NOTE: skill BOOKS are the
; BOOK type (not ALCH) and do NOT fire this event.
;
; The handler fires for ANY actor that uses an aid item, so it is guarded to the player via
; GetSelf == Player (request: player-only).
; Signature: SetOnUseAidItemEventHandler handlerScript setORremove itemOrList

ref rAidH = (begin function {ref rFnvAid}
    if eval (GetSelf == Player)
        Player.AuxVarSetRef "FNVLink_evactor" rFnvAid
        RunBatchScript "FNVLink/evt_aid_use.txt"
    endif
end)

printc "FNVLink: registering aid_use handler (189 vanilla ingestibles)."
